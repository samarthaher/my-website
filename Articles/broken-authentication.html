🔓 Broken Authentication

Common Issues in Authentication & Session Management

Common Issues

  • ✅ Weak password policies allowing users to choose easily guessable or commonly used passwords. 🔑
  • ✅ Lack of multi-factor authentication (MFA). 🔐
  • ✅ Insecure storage of passwords. 🔄
  • ✅ Predictable login credentials or tokens. 🎭
  • ✅ Vulnerabilities in authentication mechanisms. 🕵️‍♂️

🕵️ Improper Session Management

  • ✅ Predictable session identifiers or tokens. 🔢
  • ✅ Session fixation attacks. 🎭
  • ✅ Session expiration and timeout issues. ⏳
  • ✅ Insecure client-side session data. 🍪
  • ✅ Insecure or missing logout mechanisms. 🔚

🔥 Bug Bounty Checklist

🔓 1. Weak Password Policies & Brute-Force Attacks

  • ✅ Strong password policies (12+ characters, symbols, etc.) 🔑
  • ✅ Prevent weak passwords (admin123, password1) 🚫
  • ✅ Lockout after multiple failed attempts 🔒
  • ✅ CAPTCHA & rate limiting 🛑
  • ✅ Prevent username enumeration 🕵️‍♂️

🕵️ 2. Session Hijacking & Fixation

  • ✅ HttpOnly; Secure; SameSite=Strict 🍪
  • ✅ Regenerate tokens after login ♻️
  • ✅ Expire inactive sessions ⏳
  • ✅ Invalidate tokens after logout 🔚

🔀 3. MFA Bypass

  • ✅ Rate limits for OTP entry 🔢
  • ✅ OTPs expire after first use ⏳
  • ✅ Use TOTP (e.g., Google Authenticator) 📲
  • ✅ Prevent OTP brute-forcing 🚨
  • ✅ Rotate tokens after MFA login 🔄

🔐 4. OAuth & API Authentication Security

  • ✅ Whitelist OAuth redirect_uri 🔗
  • ✅ Use PKCE to prevent token hijacking 🏆
  • ✅ Limit OAuth scopes ⚠️
  • ✅ Enforce authentication on API endpoints 🔐
  • ✅ Prevent IDORs 🕵️

📂 5. Password Reset & Logout Security

  • ✅ Email/phone verification 📩
  • ✅ Reset token expires after first use 🔄
  • ✅ Invalidate all sessions after reset 🔚
  • ✅ Full session revocation on logout 🏁

🔓 6. Authentication Bypass Techniques

  • ✅ Parameter Manipulation 🔄
  • ✅ Session Reuse 🕵️
  • ✅ Cookie Modification 🍪
  • ✅ URL Tampering 🚪
  • ✅ HTTP Method Manipulation ⚡
  • ✅ Authentication Logic Flaws 🎭

🧪 Testing Scenarios

  • ✅ Default Credentials, Brute Force, Complexity, Session Mgmt 🔄
  • ✅ Password Reset, MFA, Timeout, Logout 🔐
  • ✅ Error Handling, Third-Party Auth, Timing Attacks ⏱️
  • ✅ CSRF, Race Conditions, Business Logic, Zero-Days 🚨
  • ✅ JWT, Hidden Endpoints, Token Reuse, Multi-Session 📱
  • ✅ Forgotten Password Bypass, Header Manipulation, Expiry Analysis 🔄

🎯 Final Bug Bounty Strategy

  • 🔹 Login & Auth Testing → Weak passwords, brute force
  • 🔹 Session Mgmt Testing → Hijacking, fixation
  • 🔹 MFA Implementation Testing
  • 🔹 OAuth & API Security Testing
  • 🔹 Business Logic Bypass
  • 🔹 Privilege Escalation & Admin Access
